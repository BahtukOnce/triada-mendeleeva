## 1. Текущая экосистема (что заменяем и мигрируем)

### 1.1. Сайт (index.html + auth.php + Apps Script)
- Вкладки: Дни, +Новый день (админ), Рейтинг, Логи (админ); протокол вечера с таймером ведущего.
- auth.php: MySQL `bahtukai_triada` (admins/users + sessions, password_hash, токены). ⚠ Сейчас отвечает «Access denied» — пароль БД в конфиге не подходит, авторизация на проде не работает.
- Данные игр: Google Таблица через Apps Script (`getSheets`, `getDay`, `getRating`, POST-сохранение).
- Деплой: крон `~/deploy.sh` (git pull). Рядом крон `start_gunicorn.sh` каждые 5 мин — Python-приложение (ТГ-бот).

### 1.2. Основная таблица (рейтинг) — 35 листов
- **Листы игровых дней** (~22 шт.) и **турнирные листы** (6 шт., бывают по 2–3 стола).
  Блок 15 строк = 1 игра: № игры, судья; 10 игроков: место, ник, 4 фола, 2 техфола, роль, «+», «−», Ci, Итог, протоколы, мнение, порядок выставления; ПУ (место) + ЛХ (3 кандидата); победа (Чёрные/Красные/ничья).
- **Рейтинг**: ~Σ (или ~Σ×Σ), Σ, Σ+, ПУ, ЛХ, Допы, −, Ci, Общ + статистика по ролям; выбор включённых дней; сортировки.
- **Технический лист**: агрегаты, бан-лист, порог минимума игр.
- **Экзамены/Предупреждения** — вне скоупа нового сайта (данные сохраним в архиве таблицы).

### 1.3. Техническая таблица (формы)
- «Регистрация в клуб» (318 анкет: ФИО, ник, статус, факультет, группа, опыт, ТГ, ДР) — источник профилей игроков.
- Остальные листы (записи на дни 2023–24, экзамен, опросы, генератор времени) — не переносим; генератор времени реализуем в этапе ТГ-бота/записи.

### 1.4. Система подсчёта (расшифрованные формулы)

Обозначения: K = «+» (допы), L = «−» (штрафы); победа: «Красные» (мирные+шериф) / «Чёрные» (мафия+дон) / «ничья».

**Итог игрока за игру:**
```
Итог = победный_балл + K − L + Ci + ЛХ_бонус_ПУ − фол_штраф − тех_штраф
  победный_балл = 1, если команда игрока выиграла (ничья — никому)
  ЛХ_бонус_ПУ   = бонус ЛХ игры, если игрок был ПУ и он мирный/шериф
                  (при ничьей — ПУ получает бонус независимо от роли)
  фол_штраф     = 0.6 при всех 4 фолах
  тех_штраф     = 0.3 × количество техфолов
```

**Бонус за лучший ход**: из 3 кандидатов угадано мафов 1/2/3 → 0.1/0.3/0.6.

**Ci — компенсация первоубиенному** (мирный/шериф, при заполненном ЛХ):
```
база = ПУ_счётчик_игрока / количество_игр_игрока   (по текущим агрегатам)
Чёрные выиграли:  Ci = MIN(0.4, если игр×0.4 ≥ ПУ_счётчик: ПУ_счётчик×0.4/(игры×0.4), иначе 0.4)
Красные выиграли: Ci = половина от того же
```
Решение №5: Ci пересчитывается при каждом изменении данных по всем играм (как в таблице).

**Агрегаты рейтинга**: Σ, Σ+, счётчики ПУ/ЛХ/Допы/−/Ci, ~Σ = Σ/игры (при минимуме игр, иначе «—»), сортировка «по принципу клуба» = ~Σ×Σ (альтернативы: ~Σ, по минусам), винрейт общий и по 4 ролям «57/95 (60 %)», количество игр и средний балл по ролям.

## 3. Модули сайта

### 3.0. Навигация
```
[Лого] Главная | Новости | Игры | Турниры | Рейтинг | Игроки    [🌓] [Войти / ник-бейдж ▾]
```
У админов и главы — красный бейдж ника. Дропдаун: Личный кабинет · Мои игры · Админка (judge+) · Выйти.

### 3.1. Публичная часть
- **Главная**: анонс ближайшего вечера (дата/место, записавшиеся, кнопка записи), последние новости, цифры клуба, «О клубе», блок «Администрация клуба», соцсети.
- **Новости**: лента + страница новости.
- **Игры**: список вечеров по статусам; страница вечера: записавшиеся / полные протоколы игр.
- **Турниры**: карточки (название, дата, место, статус, столы, участники); страница: описание, участники, игры по столам, итоговая таблица турнира.
- **Рейтинг**: селектор рейтинга (Общий / созданные периоды / турнирные), все колонки таблицы, сортировки (принцип клуба / ~Σ / минуса), порог минимума игр.
- **Игроки**: таблица с поиском; **профиль**: аватар, ДР, статистика всех колонок, винрейты по ролям, история игр со ссылками на протоколы, титулы турниров.

### 3.2. Личный кабинет
Аватар (загрузка с пережатием), ник, пароль, личные данные (ФИО, ТГ, ДР, факультет, группа), запись на ближайший вечер (окно «с…до…»), мои игры и статистика.

### 3.3. Админка
- **Вечер**: создать, открыть/закрыть запись, сводка записавшихся + расчёт оптимального времени (порт «генератора»: старт, пик без ведущих, окно второго стола), рассадка по столам, назначение судей.
- **Протокол игры** (порт текущего UX): рассадка 1–10, роли, фолы/техфолы, ±допы шагом 0.1, ПУ, тройка ЛХ, победитель, протоколы/мнение, порядок выбытия; таймер (15/20/30/45/60, +30, ЛХ, протокол, стоп); живой автоподсчёт; валидации; автосохранение черновика.
- **Игроки**: создать без аккаунта, привязать аккаунт к нику, слияние дублей, бан.
- **Турниры**: CRUD, участники, столы.
- **Рейтинги**: создать рейтинг (название, период), включение/исключение дней.
- **Новости**: CRUD.
- **Роли** (только глава): назначение/снятие админов и судей, передача главы.
- **Логи**: действия всех ролей.
- **Настройки**: пороги, сортировка по умолчанию, тексты главной.

## 4. Миграция данных

1. Выгрузка обеих Google-таблиц (xlsx-экспорт, уже проверено).
2. **Нормализация ников** (дубли: «НЕ_ЛИС»/«НЕ_ЛИСА🦊», «Бант»/«Бант.», «улитка»/«Улитка», «Вспышка␣»…): автомаппинг + файл ручных соответствий → отчёт на проверку.
3. Импорт: игроки (анкеты «Регистрация в клуб»: ФИО, ТГ, ДР, факультет) → игровые дни и игры основной таблицы → турниры.
4. Аккаунты: с нуля (см. открытый вопрос); игрок привязывает ник при регистрации (подтверждение админом или код от админа).
5. **Контрольная сверка**: PHP-движок пересчитывает рейтинг с нуля → автодифф с листом «Рейтинг»; расхождения разбираем поимённо.
6. Google-таблицы остаются архивом; Apps Script отключаем после приёмки этапа 4.

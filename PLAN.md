# План разработки сайта triada-mendeleeva.ru

Клубная платформа спортивной мафии «Триада Менделеева» (РХТУ). Функциональный ориентир — mafiauniverse.org, адаптированный под один клуб. Старый код (монолитный index.html + auth.php + Google Apps Script) полностью заменяется.

---

## 1. Текущая экосистема (что заменяем и мигрируем)

### 1.1. Сайт (index.html + auth.php + Apps Script)
- Вкладки: Дни, +Новый день (админ), Рейтинг, Логи (админ).
- Ведение протокола вечера с таймером ведущего (15/20/30/45/60 сек, +30, ЛХ, протокол).
- Аккаунты: ник + пароль, токены, роли user/admin (auth.php на сервере, вне git).
- Данные: чтение/запись Google Таблицы через Apps Script (`getSheets`, `getDay`, `getRating`, POST-сохранение дня).

### 1.2. Основная таблица (рейтинг) — 35 листов
- **Листы игровых дней** (~22 шт., «6 июня» и т.п.) и **турнирные листы** (6 шт., в т.ч. по столам: «Точка кипения 2026. 1–3 стол», «Турнир победы 2026», «Женский турнир 2026», «Мужицкий турнир 2026», «Осенний кубок РХТУ 2025. Стол 1–2»).
  Формат: блоки по 15 строк = 1 игра. В блоке: № игры, судья; 10 игроков: место (1–10), ник, 4 чекбокса фолов, 2 чекбокса техфолов, роль (Мирный/Мафия/Шериф/Дон), «+» допы, «−» штрафы, Ci (формула), Итог (формула), протоколы, мнение, «порядок выставления» (1–10); строка ПУ (место первоубиенного) + Лучший ход (3 места-кандидата) + доп. комментарии; строка «Победа» (Чёрные/Красные/ничья).
- **Рейтинг**: №, Игрок, ~Σ (или ~Σ×Σ), Σ, Σ+, ПУ, ЛХ, Допы, −, Ci, Общ + винрейты Мир/Маф/Шер/Дон. Панель управления: Допуск к играм (из Экзаменов), Сортировка («принципу клуба» = ~Σ×Σ / «количеству минусов» / др.), Рейтинг/Зачёт, выбор игровых дней (чекбоксы листов).
- **Технический лист**: агрегаты по игрокам (среднее, суммы, счётчики, винрейты и средние по ролям), бан-лист, «фиксированное количество игр для рейтинга» (порог для среднего), «для номинаций» (15), ссылка на техническую таблицу.
- **Экзамены**: ник, баллы (из 37), допуск Да/Нет.
- **Предупреждения**: ник + причины (до 2).
- **Шаблон** — шаблон листа дня.

### 1.3. Техническая таблица (формы) — 31 лист
- **Регистрация в клуб** (318 анкет): ФИО, игровой ник, статус (студент/…), факультет, группа, опыт, ник ТГ, как узнал, дата рождения, VK.
- **Регистрация на турниры**: ФИО, статус, фото (ссылка на Drive), ник.
- **Экзамен на знание правил** (189 попыток): баллы из 37, ответы по всем вопросам.
- **Давайте знакомиться** (волонтёрские навыки), **Рефлексия 2025** (годовой опрос).
- **Генератор оптимального времени**: по окнам «с…до…» из записи считает оптимальный старт, пик участников без ведущих, период второго стола.
- **Листы записи на дни** (15.02, 08.02, …, 2023–2025): ник, окно времени, чекбокс «зарегистрировался на mafiauniverse (клуб 154)», музыкальные пожелания.
- **Откровения Триады** — анонимные признания.

### 1.4. Система подсчёта (расшифрованные формулы)

Обозначения: K = «+» (допы), L = «−» (штрафы), победа — «Красные» (мирные+шериф) / «Чёрные» (мафия+дон) / «ничья».

**Итог игрока за игру:**
```
Итог = победный_балл + K − L + Ci + ЛХ_бонус_ПУ − фол_штраф − тех_штраф
  победный_балл = 1, если команда игрока выиграла (ничья — никому)
  ЛХ_бонус_ПУ   = N14 игры, если игрок был ПУ и он мирный/шериф
                  (при ничьей бонус даётся ПУ независимо от роли)
  фол_штраф     = 0.6, если у игрока все 4 фола
  тех_штраф     = 0.3 × количество техфолов
```

**Бонус за лучший ход (ЛХ)** — у первоубиенного 3 кандидата-места:
угадано мафов 1 → 0.1, 2 → 0.3, 3 → 0.6.

**Ci — компенсация первоубиенному** (только мирный/шериф, только если ЛХ заполнен):
```
доля = счётчик_ПУ_игрока(Рейтинг) / количество_игр_игрока(Техлист)
Чёрные выиграли (красный ПУ проиграл):  Ci = MIN(0.4, если игр×0.4 ≥ ПУ: ПУ×0.4/(игры×0.4), иначе 0.4)
Красные выиграли:                        Ci = половина от вышеуказанного
```
⚠ Особенность: в таблице Ci ссылается на ТЕКУЩИЕ агрегаты, т.е. Ci старых игр «плывёт» при добавлении новых дней. На сайте предлагается фиксировать Ci в момент завершения игры (см. вопрос №5).

**Рейтинг (агрегаты):**
- Σ — сумма итогов; Σ+ — сумма «плюсовых» составляющих; счётчики ПУ, ЛХ, Допы, −, Ci.
- ~Σ — средний балл = Σ / игры (если игр ≥ порога, иначе «—»).
- Сортировка «по принципу клуба» = ~Σ × Σ; альтернативы: ~Σ, «по количеству минусов».
- Винрейт общий и по ролям: «57/95 (60 %)»; количество игр и средний балл по ролям.
- Допуск к играм: экзамен сдан (≈ ≥35 из 37) → «Да»; бан-лист исключает из рейтинга.

## 3. Модули сайта

### 3.1. Публичная часть
- **Главная**: анонс ближайшего вечера (дата/время/место, счётчик записавшихся, кнопка «Записаться»), новости, цифры клуба (игроков, игр, сезонов), блок «О клубе», соцсети, дни рождения (опц.).
- **Игроки**: таблица (аватар, ник, игр, ~Σ, Σ, винрейт), поиск, сортировки.
- **Профиль игрока**: аватар, статистика всех колонок рейтинга, винрейты по ролям с графикой, история игр (дата, роль, итог, ссылка на протокол), достижения/титулы турниров, допуск к играм. Личные данные — скрыты.
- **Вечера**: список с статусами; страница вечера: записавшиеся (для открытых) / все игры с полными протоколами (для завершённых).
- **Турниры**: карточки (как на mafiauniverse: название, дата, место, статус, участники), страница: описание, регистрация, столы, результаты, итоговая таблица.
- **Рейтинг**: все колонки таблицы; переключатели: сезон/период (выбор дней), сортировка (принцип клуба / ~Σ / минуса), рейтинг/зачёт; подсветка недопущенных.
- **Экзамен**: онлайн-тест из текущих вопросов формы, автопроверка, результат и допуск сразу в профиль.

### 3.2. Личный кабинет
- Профиль: аватар (загрузка), ник, пароль, личные данные (ФИО, ТГ, ДР, факультет…).
- Запись на ближайший вечер: окно «с…до…», музыкальные пожелания, отмена записи.
- Мои игры и статистика, мои попытки экзамена, мои предупреждения.

### 3.3. Админка
- **Вечер**: создать (дата, место), открыть/закрыть запись; сводка записи с «генератором времени» (порт формул: оптимальный старт, пик без ведущих, окно второго стола); рассадка по столам; назначение судей.
- **Протокол игры** (перенос текущего UX + улучшения): рассадка 1–10 (автоподстановка из записи), роли, фолы/техфолы чекбоксами, ±допы с шагом 0.1, ПУ, тройка ЛХ, победитель, протоколы/мнение, порядок выставления/отстрела, комментарий; таймер ведущего (15/20/30/45/60, +30, ЛХ, протокол, стоп); живой автоподсчёт итогов; валидации перед сохранением (как сейчас: «ЛХ не заполнен при непромахе» и т.п.); черновики (autosave).
- **Игроки**: создать гостя, привязать аккаунт к нику, слияние дублей-ников, предупреждения, бан, ручной допуск.
- **Турниры**: CRUD, регистрация, столы.
- **Экзамен**: CRUD вопросов, порог, просмотр попыток.
- **Новости**: CRUD.
- **Логи**: все действия админов/судей.
- **Настройки**: пороги, сортировка по умолчанию, текущий сезон, тексты главной.

### 3.5. Безопасность
`password_hash`/`password_verify`, сессии в httponly+secure cookie, CSRF-токены на всех формах, prepared statements везде, rate-limit на логин/регистрацию, проверка прав в каждом эндпоинте, валидация загрузок (аватары: тип/размер/пережатие), логирование действий.

## 4. Миграция данных

1. Выгрузка обеих Google-таблиц (xlsx-экспорт по публичным ссылкам).
2. **Нормализация ников** — в таблицах есть дубли/варианты («НЕ_ЛИС» / «НЕ_ЛИСА🦊» / «НЕ_ЛИС 🦊», «Бант» / «Бант.», «улитка» / «Улитка», «Вспышка␣»…). Скрипт строит автомаппинг (trim, регистр, эмодзи) + файл ручных соответствий → отчёт на проверку глазами.
3. Импорт: игроки (анкеты «Регистрация в клуб») → дни и игры основной таблицы (значения, не формулы) → турнирные листы → экзамены (все попытки) → предупреждения, бан-лист.
4. Перенос аккаунтов из текущего auth.php (формат хранилища — вопрос №2).
5. **Контрольная сверка**: движок рейтинга на PHP пересчитывает всё с нуля → автоматический дифф с колонками листа «Рейтинг»; расхождения разбираем поимённо.
6. Старая история 2023–2024 из техтаблицы (только записи, без протоколов) — см. вопрос №13.

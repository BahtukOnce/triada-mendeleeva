Ты подключаешься к разработке сайта клуба спортивной мафии «Триада Менделеева»
(https://triada-mendeleeva.ru). Эта команда нужна, чтобы **спокойно синхронизироваться**
между разными ПК и понять текущее состояние, прежде чем что-то менять. Работай кратко и по делу.

## 1. Синхронизация git
- `git fetch --all --prune`
- Покажи текущую ветку и насколько она ahead/behind `origin`.
- `git status` — есть ли несохранённые правки.
- Последние коммиты: `git log --oneline -10 origin/main` и (если есть) `origin/test`.
- Реши по ситуации и **предложи действие, не выполняя его молча**:
  - локально чисто и ветка отстаёт → предложи `git pull --ff-only`;
  - есть локальные несохранённые/непушнутые правки → предупреди и НЕ делай pull/reset, спроси, что с ними делать;
  - ветки разошлись → опиши расхождение и спроси стратегию.
- Никаких `reset --hard`/`push --force` без явного согласия.

## 2. Контекст проекта (прочитай, держи в голове)
- `PLAN.md` — план этапов и статус.
- `README.md` — обзор.
- `config.example.php` — какие ключи нужны в `config.php` (сами секреты в репозитории НЕ хранятся; они только в `config.php` на сервере).
- Структура:
  - `public_html/` — DocumentRoot (страницы, `assets/`, `admin/`, `api/`).
  - `inc/` — `bootstrap.php`, `db.php`, `auth.php`, `helpers.php`, `layout.php`, `rating.php`, `elo.php`, `bot_lib.php`.
  - `db/migrations/*.sql` — нумерованные идемпотентные миграции.

## 3. Деплой (напомни, не запускай без задачи)
- Стек: PHP 8.2 + MySQL (PDO), хостинг Beget. Тёмная тема, CSS-переменные `--bg/--sf/--tx/--ac`.
- Ветки: `main` = прод, `test` = разработка. В `test` пушим свободно; в `main` — после приёмки на тесте.
- Деплой: push → GitHub webhook тянет на сервер. Ручной деплой — подписанный (HMAC-SHA256) POST на `/deploy.php` с заголовком `X-Hub-Signature-256`, секрет `deploy_secret` берётся из `config.php` на сервере (в репо его нет).
- Миграции применяются при деплое автоматически. `/migrate.php?key=<deploy_secret>&elo=1` — пересчёт ELO; `&cleannicks=1` — чистка эмодзи из ников.
- Кэш ассетов: при правке `style.css`/`app.js` поднимай `?v=N` в `inc/layout.php`.

## 4. Правила разработки
- Точечные правки, не переписывать лишнее; код в стиле окружающего.
- Уникальные DOM-id; JS — в IIFE и проверять `node --check` перед деплоем.
- После правок убеждаться, что страница парсится (нет 500) и счётчики/числа сходятся.
- Коммиты осмысленные; не пушить секреты (config.php, токены, пароли БД).

## 5. Отчёт
В конце дай короткую сводку: на какой ветке, синхронизировано ли с origin, есть ли несохранённое,
что сейчас в работе (из `PLAN.md` и последних коммитов) — и спроси, что делаем.
